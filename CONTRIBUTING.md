# Contributing Guidelines

## How to Contribute

This project is open to contributions! Whether you find bugs, want to add features, or improve documentation, we'd love your help.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages: `git commit -m "Add feature: description"`
7. Push to your fork
8. Open a Pull Request

## Code Style

- Use Python 3.8+
- Follow PEP 8 style guide
- Use meaningful variable and function names
- Add docstrings to functions

## Testing

Before submitting a PR:
1. Test on sample data
2. Verify outputs match expected format
3. Check memory usage for large datasets
4. Ensure no hard-coded paths

## Bug Reports

When reporting bugs, include:
- Python version
- Dependencies versions
- Minimal reproducible example
- Expected vs actual behavior
- Full error traceback

## Feature Requests

Describe:
- The problem you're solving
- Proposed solution
- Why this would be useful

## Documentation

- Update README.md if adding features
- Add docstrings to new functions
- Include usage examples
